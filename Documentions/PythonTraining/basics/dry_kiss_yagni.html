Day 4 — Clean Code
DRY  ·  KISS  ·  YAGNI
"These aren't rules — they're instincts.
You develop them by noticing the same mistakes repeatedly."
Principle ❌  Violation ✅  Fix
DRY Don't Repeat Yourself
⚠ duplicated logic
def fine_for_book(days): return 0.20 * days def fine_for_dvd(days): return 0.20 * days # copy-paste
Same formula in 3 places → change the rate, hunt every copy.
✓ single source of truth
def calculate_fine(item): return item.daily_fine * item.overdue_days # one place to change, every caller benefits
The rate lives on the item. The formula lives once.
KISS Keep It Simple
⚠ overcomplicated
class TitleSorter: def __init__(self, strategy, locale, reverse): ... # 60 lines for a sort
A custom sort engine for 12 book titles.
✓ simplest thing that works
items.sort(key=lambda i: i.title) # could a newcomer read this in 30 s? # yes → ship it
The standard library already solved this.
YAGNI You Aren't Gonna Need It
⚠ speculative code
def export_to_xml(self): ... # "just in case" def export_to_yaml(self): ... # never been called
No ticket, no user, no requirement. Just future-guessing.
✓ build when needed
def export_to_csv(self): ... # real ticket, real user # xml/yaml: add them when asked
Every line of unused code is a line to maintain, debug, and explain.
DRY — ask yourself
If I change this in one place, does it break somewhere else?
KISS — ask yourself
Could a newcomer understand this in 30 seconds?
YAGNI — ask yourself
Is there a real ticket for this, or am I guessing the future?
